## 1. 打开调试台

调试台是一个独立浮动窗口，可从主窗口打开：

- **标题栏** → 帮助 ▼ → 🔧 调试台
- 调试台独立于主窗口之外，可随意拖拽和缩放
- 窗口右上角 `✕` 关闭后，后台日志继续运行，再次打开自动恢复

## 2. 命令系统概述

### 命令结构

所有命令使用**空格**分隔节点，形成树形结构。完整命令树定义在 `config/command_defs.json`：

```
命令集 → 子命令集（可多层嵌套）→ 后缀（具体参数值 / <占位参数>）
```

例如：

```
log level DEBUG
│    │      └─ 后缀：具体参数值
│    └──────── 子命令集：日志级别
└───────────── 命令集：日志管理
```

### 命令格式

- **标准输入格式**：空格分隔（如 `log level DEBUG`、`plugin list`）
- **代码内部格式**：冒号分隔（如 `log:level`、`plugin:list`）
- 输入时使用空格格式，系统自动转换

### 别名

可以通过 `alias` 命令设置简短别名，提高效率：

```
alias set ll "log level"      # 之后输入 ll DEBUG 等价于 log level DEBUG
alias list                      # 查看所有别名
alias remove ll                 # 删除别名
```

别名存储在 `config/aliases.json`，重启后保留。

## 3. 命令分类速查

| 命令 | 说明 |
|------|------|
| `help` | 显示所有可用命令列表（含分类和参数说明） |
| `clear` | 清空当前日志显示区 |

| `alias set <别名> <命令>` | 设置命令别名 |
| `alias remove <别名>` | 删除别名 |
| `alias list` | 列出所有已设置别名 |

| 命令 | 参数 | 说明 |
|------|------|------|

## 4. 智能补全

调试台支持**两阶段补全**（Minecraft 式）：输入命令名匹配 → Tab/空格 → 参数补全。

### 阶段一：命令名匹配

输入部分命令名时，弹出匹配的命令列表：

```
输入: l
弹出: log  日志管理
      ...（其他匹配项）
```

- 按 **Tab** 填充当前选中的命令
- 按 **空格** 填充并进入参数补全模式
- 输入**为空时**自动弹出全部可用命令列表（按分类分组）

### 阶段二：参数补全

命令名+空格后自动切入参数补全模式，显示该命令的参数可选值：

```
输入: log level _
弹出: DEBUG
      INFO
      WARN
      ERROR
```

参数补全支持三种类型（通过 `SuggestPopup.ARG_DEFS` 注册表定义）：

| 类型 | 说明 | 颜色 | 可选性 | 示例 |
|------|------|------|--------|------|
| `static` | 固定可选值列表 | 青色 `#89b4fa` | 可选中 | `log level` → `DEBUG`/`INFO`/`WARN`/`ERROR` |
| `dynamic` | 运行时动态拉取 | 青色 | 可选中 | `plugin info` → 实时从系统拉取插件 ID |
| `open` | 自由输入占位提示 | 灰色 `#6c7086` 斜体 | 不可选中 | `timer start` → `<秒数>` |

### 补全说明

- 命令名用**主色**（`#cdd6f4`），提示文字用白色
- 动态参数（如 `<插件ID>`）会实时从系统拉取列表
- 占位参数（如 `<秒数>`）显示为灰色不可选
- 插件注册的自定义命令也自动集成到补全系统中

### 补全注册表

参数补全信息定义在 `suggest_popup.py` 的 `ARG_DEFS` 字典中：

```python
ARG_DEFS = {
    "log:level": {
        "static": ["DEBUG", "INFO", "WARN", "ERROR"],
        "dynamic": None,
        "open": None,
    },
    "plugin:info": {
        "static": None,
        "dynamic": "plugin_list",  # 运行时拉取
        "open": "<插件ID>",
    },
    ...
}
```

完整命令树定义在 `config/command_defs.json`（树形结构，空格分隔节点），`ARG_DEFS` 需与之一致。

## 5. 键盘快捷键

| 快捷键 | 说明 |
|--------|------|
| `↑` | **输入框空时**：浏览上一条历史命令 |
| `↓` | **输入框空时**：浏览下一条历史命令 |
| `Tab` | 自动补全当前输入的词 |
| `Enter` | 执行当前命令 |
| `Ctrl+F` | 切换搜索栏（在输出区搜索） |
| `Esc` | **搜索栏激活时**：关闭搜索栏 |
| `Ctrl+L` | 清空日志显示区 |

### 历史命令

- 所有执行过的命令自动保存到历史（最多 50 条）
- 输入 `!!` 重复上一条命令
- 通过 `↑` / `↓` 在历史中前后浏览

### 补全键

- **悬浮框可见时**：`↑↓` 在补全选项间切换，`Tab` 选中填充
- **悬浮框隐藏时**：`↑↓` 浏览历史命令

## 6. 搜索输出

### 快捷键

`Ctrl+F` 打开搜索栏。

### 使用

1. 按 `Ctrl+F` 显示搜索栏
2. 输入关键字并按 `Enter`
3. 所有匹配项以黄色高亮显示
4. 自动定位到第一个匹配项
5. 按 `Esc` 关闭搜索栏并清除高亮

## 7. 输出操作

### 右键菜单

在输出区任意位置右键：

| 操作 | 说明 |
|------|------|
| 📋 复制选中 | 复制当前选中的文本 |
| 📋 复制全部 | 复制输出区全部内容 |
| 💾 保存到文件... | 将输出保存到 `docs/log/debug_output_saved.txt` |

### 命令执行耗时

每条命令执行后自动追加耗时信息：

```
> log level DEBUG
  ✅ 12ms         ← 灰色小字，自动追加
```

## 8. 别名管理

### 设置别名

```
alias set ll "log level"        # 之后 ll DEBUG → log level DEBUG
alias set p "plugin list"        # 之后 p → plugin list
alias set c "check all"          # 之后 c → check all
```

### 查看别名

```
alias list
```

输出示例：
```
═══ 已设置别名 ═══
  ll  →  log level
  p   →  plugin list
共 2 个别名
```

### 删除别名

```
alias remove ll
```

### 文件存储

别名保存在 `config/aliases.json`，格式：

```json
{
  "ll": "log level",
  "p": "plugin list"
}
```

## 9. 常见使用场景

### 场景 1：诊断系统问题

```
check all
```

一次性检查管理器/插件/面板加载状态，快速定位未加载组件。

### 场景 2：切换面板无需鼠标

```
panel open speech        # 打开 AI 写稿
panel open material      # 打开资料池
panel close              # 关闭面板
```

### 场景 3：监视变量变化

```
monitor enable variable
monitor enable function
monitor log view
```

先启用监视，再查看监视日志。

### 场景 4：快速切换主题

```
theme list                    # 查看可用主题
theme set notion_light        # 切换浅色主题
```

### 场景 5：使用别名提效

```
alias set ll "log level"
alias set p "panel open"
ll DEBUG
p notes
```

### 场景 6：导出分析

```
monitor log export            # 导出监视日志
config show                   # 查看配置
project info                  # 查看项目信息
```

## 10. 命令定义来源

调试台命令体系存在三层定义，增删命令时需同步更新：

| 优先级 | 文件 | 说明 |
|--------|------|------|
| **权威来源** | `config/command_defs.json` | 树形结构定义全部命令及其元信息（分类/描述/参数/动态列表） |
| **执行分派** | `workers/debug_console/command_handler.py` | `BUILTIN_COMMANDS`（静态列表）+ `execute()` 的 if-elif 分派链 |
| **参数补全** | `workers/debug_console/suggest_popup.py` | `ARG_DEFS` 注册表（static/dynamic/open 三级） |

**新增命令流程**：
1. 在 `command_defs.json` 中添加命令树节点
2. 在 `command_handler.py` 中添加 `_cmd_*` 方法 + 接入 `execute()` if-elif
3. 在 `suggest_popup.py` 的 `ARG_DEFS` 中添加参数补全定义
4. 更新本文档和 `docs/命令速查.md`
