## 一、软件简介

**StarDebate ★ 辩之星** 是一款面向中文辩论爱好者与参赛辩手的全流程桌面辅助软件。它把"备赛—写稿—质询—复盘"四个环节整合在同一个工作台中，并结合大语言模型（DeepSeek、OpenAI 兼容接口）提供 AI 辅助。

软件采用模块化架构（25 个 worker 模块 + 8 个通用组件），具备以下能力：

- **AI 写稿 / 分析 / 扩写 / 质询**：用 AI 完成繁琐的初稿与复盘工作。
- **可视化框架**：思维导图式的立论/驳论框架编辑。
- **结构化资料**：资料池、资料稿、便签统一管理。
- **实战训练**：快速刷题、立论驳论练习。
- **可扩展**：插件系统允许第三方扩展功能。
- **可定制**：6 套主题、快捷键、API 全部开放配置。

### 适用人群

| 人群 | 适用功能 |
|------|----------|
| 校园辩论队队员 | 框架搭建、AI 写稿、模拟质询、赛程管理 |
| 辩论新手 | 快速刷题、立论驳论练习、辩词素材库插件 |
| 辩论教练 | 资料库、批注、复盘分析 |
| 赛事组织者 | 赛程管理、计时器插件 |

## 二、核心特色

| 特色 | 说明 |
|------|------|
| **三文件架构** | 启动器 / 主窗口 / 日志服务分层独立，主进程崩溃不影响日志写入 |
| **6 套主题** | Catppuccin Mocha/Latte/Macchiato、Nord、Notion Dark/Light，护眼美观 |

| **插件沙箱** | 19+ 个 Plugin API + 权限检查，第三方扩展安全可控 |
| **异步 Worker** | 所有 AI 任务后台执行，界面永不阻塞 |

## 三、第一次运行

### 3.1 启动方式

直接双击 `StarDebate.py`，或使用命令行：

```bash
python StarDebate.py
```

启动后会看到：

1. **闪屏卡片**（约 1-2 秒）：显示"StarDebate ★ 辩之星"标题与呼吸光晕进度条。
2. **主窗口**：浅色默认主题（Notion Light），上次打开的项目会自动加载（如有）。

### 3.2 首次配置 API

软件的所有 AI 功能均依赖外部大模型 API。首次使用前请：

1. 点击右上角 **设置** 按钮（或顶部导航栏齿轮图标）。
2. 进入 **"API 配置"** 页面。
3. 填写：
   - **API URL**：默认 `https://api.deepseek.com/v1/chat/completions`
   - **API Key**：在对应服务商控制台申请
   - **Model**：默认 `deepseek-chat`
   - **Max Tokens**：默认 2048
   - **Temperature**：默认 0.7
4. 点击 **"测试连接"** 验证。
5. 点击 **"保存"** —— 密钥将自动加密写入 `config/api_config.json`。

> 💡 配置完成后，所有 AI 功能（写稿/分析/扩写/质询/接质）即可使用。

### 3.3 新建第一个辩论项目

1. 点击顶部 **"新建辩论"** 按钮。
2. 填写：
   - **辩题**
   - **正方观点 / 反方观点**
   - **赛制**（华辩 / 奥瑞冈 / 自定义）
   - **存储路径**（默认在 `config/stardebate_index.json` 注册的项目目录）
3. 确认后，结构树会自动展开，您可以直接在框架区开始构建立论体系。

## 五、核心功能详解

### 5.1 辩论框架

> 位置：左侧导航 → 框架

**功能**：思维导图式的立论/驳论框架编辑器，支持正方/反方双视角。

**使用步骤**：

1. 在左侧结构树中选择"正方框架"或"反方框架"节点。
2. 中央画布出现空白思维导图。
3. **添加节点**：双击空白处，输入论点名称。
4. **建立连接**：从一个节点边缘拖拽到另一个节点。
5. **节点类型**：右键节点可选择类型（论点 / 论据 / 反驳 / 类比 / 数据）。
6. **AI 助手**：点击画布上方的"AI 对话"按钮，可让 AI 帮您补全某节点的论证。

**文件存储**：`workers/framework/` 下的 JSON 文件，自动保存。

---

### 5.2 一辩稿编辑

> 位置：右侧面板 → 一辩稿

**功能**：专业的辩稿编辑器，支持 Markdown、自动保存、撤销/重做、资料绑定。

**使用步骤**：

1. 打开"一辩稿"面板，新建正方/反方辩稿。
2. 在编辑区撰写内容，支持：
   - **Markdown 语法**（标题、列表、引用、代码块）
   - **实时字数统计**（顶部状态栏）
   - **自动保存**（每 30 秒）
   - **撤销/重做**（Ctrl+Z / Ctrl+Y）
3. **词汇绑定资料**：选中某个关键词，会出现悬浮卡片提示，点击"绑定资料"可关联资料池中的素材，悬浮卡显示 `has_material.svg` 图标。
4. **导出**：右上角"导出"按钮，可导出为 `.md` 或嵌入到 `.stardebate` 项目。

---

### 5.3 AI 写稿

> 位置：右侧面板 → AI 写稿

**功能**：基于辩题与框架，一键生成一辩稿初稿。

**使用步骤**：

1. 切换到 "AI 写稿" 面板。
2. 选择立场（正方/反方）和辩题（自动从当前项目读取）。
3. 设置：
   - **风格**（严谨 / 犀利 / 文艺）
   - **字数**（建议 600-1000 字）
   - **是否引用资料池素材**（勾选后 AI 会参考已导入的资料）
4. 点击 **"生成"** —— 进度条显示，后台异步执行。
5. 完成后会自动填充到一辩稿编辑器，可继续手动修改。

> ⏱ 大约需要 10-30 秒，取决于 API 速度。

---

### 5.4 AI 分析

> 位置：左侧导航 → AI 分析（或中部工作区）

**功能**：对辩题、正反方观点、已有辩稿进行深度分析，识别论证强弱项、逻辑漏洞。

**使用步骤**：

1. 打开"AI 分析"页面。
2. 选择分析对象：
   - **辩题分析**（输入辩题，AI 给出正反双方潜在论据）
   - **辩稿分析**（粘贴辩稿，AI 评估论证质量、找出漏洞）
3. 点击 **"开始分析"**。
4. 报告会分为：
   - 核心论点评估
   - 论证结构评分
   - 潜在反驳预演
   - 改进建议
5. 报告可保存为 `analysis_pro.json` / `analysis_con.json`。

---

### 5.5 AI 扩写

> 位置：右侧面板 → AI 扩写

**功能**：选中一段简短论述，让 AI 自动扩展为完整论证。

**使用步骤**：

1. 在一辩稿编辑器中选中需要扩写的文本。
2. 切换到 "AI 扩写" 面板（自动带入选中文本）。
3. 选择扩写方向（更详细 / 加入类比 / 补充数据 / 强化论证）。
4. 点击 **"扩写"**。
5. 完成后弹出对比窗口，可选择"替换原文"或"插入到光标位置"。

---

### 5.6 模拟质询 & 接质

> 位置：左侧导航 → 模拟质询 / 模拟接质

#### 模拟质询

**功能**：AI 扮演对方辩手，向您提出质询问题，训练临场反应。

1. 选择立场（您要扮演正方还是反方）。
2. AI 生成问题 → 您在编辑区回答 → AI 评估回答质量（逻辑性、针对性、说服力）。
3. 持续多轮，可随时结束并查看汇总报告。

#### 模拟接质

**功能**：您提出的问题，AI 扮演对方辩手回答，训练提问技巧。

1. 输入您的问题 → AI 回答 → AI 同时给出"提问质量评分"（是否刁钻、是否直击要害）。
2. 结果保存到 `cross_exam.json` / `accept_exam.json`。

---

### 5.7 模拟训练

> 位置：右侧面板 → 训练

训练面板包含两个子模块：

#### 5.7.1 快速刷题

选择题模式的快速刷题：

1. 加载题库（可使用内置题库或自定义题库）。
2. 逐题作答：4 个选项按钮，点击即答。
3. 立即反馈：正确/错误 + 解析。
4. 统计：本次正确率、所用时间。
5. 训练记录保存到 `training_*.json`。

#### 5.7.2 立论驳论练习

结构化练习模板：

1. 选择题型（立论 / 驳论 / 总结陈词）。
2. 输入辩题与立场。
3. 在规定时间内完成写作（可设置 5/10/15 分钟倒计时）。
4. AI 评分并给出改进建议（论证完整性、逻辑严密性、语言表达）。

---

### 5.8 资料池

> 位置：左侧导航 → 资料池

**功能**：v4.0.0 新增，集中管理所有辩论参考资料。

**支持格式**：PDF、Word、TXT、Markdown、图片（OCR 识别文字）。

**核心操作**：

| 操作 | 说明 |
|------|------|
| 导入文件 | 拖拽到资料池或点击"导入"按钮 |
| 全文搜索 | 本地 BM25 + AI 混合搜索 |
| 查看原文 | 双击文件卡片预览 |
| 引用到辩稿 | 选中关键词 → 绑定弹窗 → 关联到一辩稿 |
| **AI 综合总结导出** | 一键对所有文件生成 AI 总结并合并为 MD |

**AI 总结导出**：

1. 左侧面板点击 **"🤖 AI 总结导出"** 按钮。
2. 进度面板显示当前处理的文件名与整体进度。
3. 每个文件生成 500+ 字摘要、关键要点、核心内容、相关话题。
4. 完成后合并为结构化 Markdown 文件，可保存到任意位置。

---

### 5.9 资料稿管理

> 位置：左侧导航 → 资料稿

**功能**：分类管理辩论素材与参考文献。

**操作**：

- 创建分类（理论 / 数据 / 案例 / 类比）
- 添加条目（标题 + 内容 + 来源链接 + 标签）
- 标签筛选 / 全文搜索
- 一键复制到剪贴板
- 导出为 Markdown

**文件存储**：`ref_doc_pro.json` / `ref_doc_con.json`。

---

### 5.10 便签

> 位置：右侧面板 → 便签

**功能**：浮动便签工具，记录临时想法、关键论点、反驳要点。

**使用**：

- 点击"+ 新建便签"创建一张便签。
- 便签可自由拖拽、改变大小、设置背景色。
- 数据持久化到 `sticky_notes.json`，关闭软件后再次打开仍在。

---

### 5.11 赛程管理

> 位置：左侧导航 → 赛程

**功能**：自定义赛制、编排赛程、管理对阵。

**支持的赛制**：

- 华语辩论赛（陈词 → 质询 → 自由辩 → 总结）
- 奥瑞冈赛制
- 英国议会制（BP）
- 自定义（自由配置环节与时间）

**操作**：

1. 创建赛事，填写赛事名称、时间、地点。
2. 添加参赛队伍。
3. 抽签或手动编排对阵。
4. 点击"开始比赛"自动启动计时器（依赖"辩论计时器"插件）。
5. 记录胜负与最佳辩手。

---

### 5.12 结构树

> 位置：左侧导航 → 结构树

**功能**：以树形结构展示当前辩论项目的所有内容（框架、辩稿、资料、分析等）。

**操作**：

- 单击节点：右侧显示对应内容。
- 右键节点：可重命名、删除、导出。
- 拖拽节点：调整层级关系。
- 顶部搜索框：按名称快速定位。

---

### 5.13 .stardebate 项目管理

> 位置：左侧导航 → 项目树

**功能**：以项目为单位管理多个辩论主题。

**.stardebate 文件**：

- 是 StarDebate 的项目文件格式（AES + 用户密码双层加密）。
- 包含：辩题、正反方观点、框架、辩稿、资料、分析、便签的完整数据。
- 可在项目树中创建/打开/删除/导出 `.stardebate` 文件。
- 加密后跨设备携带、分享给队友都安全。

**项目树图标**：使用 SVG 主题色图标（`icon/common/` 目录），自动跟随主题色。

**自动加载上次项目**：`config/config.json` 中保存"最后打开项目路径"，下次启动自动恢复。

## 六、设置系统

点击右上角 **设置** 进入设置中心，包含 8 个子页面。

### 6.1 API 配置

- **API URL**：API 端点（默认 DeepSeek）
- **API Key**：自动加密存储
- **Model**：模型名称
- **Max Tokens**：单次响应最大 Token 数
- **Temperature**：生成温度（0=精确，1=创造）
- **测试连接**：点击验证 API 可用性

### 6.2 主题与外观

- **主题选择**：6 套主题（Catppuccin Mocha / Latte / Macchiato、Nord、Notion Dark / Light）
- **主题实时预览**：切换后立即应用
- **字体大小**：全局调整（最小 10px）
- **图标模式**：emoji / SVG

### 6.3 快捷键

- **18 个内置快捷键**（5 分类：编辑/导航/AI/插件/全局）
- **录制修改**：点击快捷键后按下新组合键即可重新绑定
- **冲突检测**：自动提示已占用
- **插件可注册自定义快捷键**

### 6.4 日志设置

- **日志级别**：DEBUG / INFO / WARN / ERROR
- **日志保留策略**：按大小 / 按时间 / 全部保留
- **崩溃自动分析**：开启/关闭

### 7.2 安装 .stp 插件

`.stp`（StarPlugin Package）是基于 Zip 的单文件插件封装格式。

**安装方式**：

1. **拖拽安装**：将 `.stp` 文件拖到主窗口，会自动弹出安装预览。
2. **按钮安装**：插件面板 → "📦 安装插件" → 选择 `.stp` 文件。

**安装流程**：

1. **格式识别**：自动识别 Zip 注释 `"StarPlugin"`。
2. **SHA256 校验**：所有文件排序后拼接计算，验证完整性。
3. **依赖检查**：声明式依赖，缺失会阻止安装。
4. **权限审查**：开发者声明的权限，用户授权。
5. **安装后默认禁用**：可在插件面板手动启用。
6. **冲突处理**：弹窗选择（默认覆盖升级）。

**卸载**：

- 插件面板 → 选中插件 → 卸载 → 弹窗选择（默认仅禁用）。

### 7.3 插件管理工具

`plugin_manager/` 是独立的 PySide6 桌面工具，用于：

- 创建新插件
- 编辑已有插件元数据

- 校验 `.stp` 完整性

另提供命令行版本 `tools/pack_stp.py`（`--validate` 校验 / `-o` 打包）。

## 九、常见问题（FAQ）

### Q1：AI 功能点击后没有反应？

**A**：请先检查 API 配置：

1. 进入 **设置 → API 配置**
2. 确认 API Key 已填写且测试连接通过
3. 检查 API 余额

### Q2：闪屏卡住，主窗口打不开？

**A**：可能是上次异常关闭导致配置损坏。尝试：

1. 删除 `config/config.json`（会重置主题与最后打开项目）
2. 重新启动

如仍无法解决，请查看日志 `docs/*.log` 获取详细错误。

### Q3：插件安装后没有出现在导航栏？

**A**：插件安装后默认禁用。请：

1. 打开 **插件面板**
2. 找到对应插件，点击"启用"
3. 重启软件生效

### Q4：.stardebate 文件忘记密码怎么办？

**A**：.stardebate 使用 AES-256 双层加密，**无法找回密码**。请妥善保管。

建议：

- 在多个安全位置备份密码
- 使用密码管理器（如 Bitwarden、1Password）
- 定期导出未加密版本作为备份

### Q5：能否同时打开多个辩论项目？

**A**：当前版本仅支持单项目窗口。如需切换：

1. 关闭当前项目（或最小化主窗口，配置会自动保存）
2. 通过项目树打开新项目

### Q6：AI 写稿生成的内容质量不佳？

**A**：可尝试：

1. **优化提示词**：在设置 → API 配置中调整 Temperature（建议 0.6-0.8）
2. **先建框架**：使用"辩论框架"先搭建清晰结构，AI 写稿会基于框架更精准
3. **引用资料**：在 AI 写稿设置中勾选"引用资料池素材"
4. **多轮迭代**：将 AI 输出作为初稿，手动修改后再用 AI 扩写

### Q7：如何备份数据？

**A**：

- **整个项目**：导出为 `.stardebate` 文件
- **快速备份**：直接复制项目目录
- **配置备份**：备份 `config/` 目录
- **日志备份**：`docs/*.log` 文件

### Q8：能离线使用吗？

**A**：除 AI 功能外，所有功能（框架编辑、辩稿撰写、资料管理、训练）均可离线使用。AI 功能需要联网调用 API。

### Q9：如何反馈问题或建议？

**A**：

- 启动"起居注"记录操作 → 导出日志
- 提交 Issue（附上日志文件与复现步骤）
- 加入用户交流群
