## 1. 基本操作

| 命令 | 参数 | 说明 |
|------|------|------|

## 2. 日志命令

| 命令 | 参数 | 说明 |
|------|------|------|
| `log:level` | `DEBUG`\|`INFO`\|`WARN`\|`ERROR` | 设置日志显示级别（不影响文件记录） |
| `log:export` | — | 导出当前会话日志到文件（弹出保存确认） |
| `log:clean` | — | 手动清理超过 7 天的旧日志文件（仅删 `debug_*.log`） |
| `log:path` | — | 显示当前会话的日志文件完整路径 |
| `log:keep` | — | **一次性标记**：正常退出时保留本次运行日志不被删除 |

### 日志级别说明

| 级别 | 说明 |
|------|------|
| `DEBUG` | 最详细，包含所有调试信息 |
| `INFO` | 常规信息（默认级别） |
| `WARN` | 警告信息 |
| `ERROR` | 错误信息 |

**示例**:
```
log:level DEBUG      # 显示所有级别的日志
log:level ERROR      # 仅显示错误日志
```

### 日志文件位置

日志自动保存在 `docs/log/` 目录下，文件名格式: `debug_YYYYMMDD_HHMMSS.log`

启动调试台时自动删除超过 **7 天** 的旧日志文件。

## 4. 插件命令

| 命令 | 参数 | 说明 |
|------|------|------|
| `plugin:list` | — | 列出所有已安装插件（名称/版本/状态） |
| `plugin:info` | `<插件ID>` | 查看指定插件的详细信息 |
| `plugin:reload` | — | 重新加载所有插件 |
| `plugin:enable` | `<插件ID>` | 启用指定插件 |
| `plugin:disable` | `<插件ID>` | 禁用指定插件 |

```

plugin:info debate_timer
  ═══ 插件详情: debate_timer ═══
  名称: 辩论计时器
  版本: 1.0.0
  作者: StarDebate
  描述: 为辩论提供计时功能
  状态: 启用
```

## 6. 计时器命令

| 命令 | 参数 | 说明 |
|------|------|------|
| `timer:start` | `<秒数>` | 启动倒计时器，每10秒播报一次，最后5秒每秒播报 |
| `timer:stop` | — | 停止当前正在运行的计时器 |

### 示例

```
timer:start 300      # 启动 5 分钟倒计时
  ⏱  倒计时开始: 05:00
  ⏱  剩余: 04:50
  ...
  ⏱  剩余: 00:05
  ⏱  剩余: 00:04
  ...
  ⏱  时间到！

timer:stop            # 手动停止计时器
  ⏱  计时器已停止
```

## 11. 快捷技巧

### 键盘操作

| 快捷键 | 说明 |
|--------|------|
| `↑` | 浏览上一条历史命令 |
| `↓` | 浏览下一条历史命令 |
| `Tab` | 自动补全命令名称 |
| `Enter` | 执行当前命令 |
| `Ctrl+L` | 清空日志显示区 |

### 其他技巧

1. **重复命令**：输入 `!!` 可重复执行上一条命令
2. **历史记录**：最多保存 50 条历史命令
3. **自动补全**：输入命令前几个字符后按 `Tab`，如 `log` → `log:level`
4. **暂停刷新**：点击「⏸ 暂停」按钮可冻结日志输出，排查问题时非常有用
5. **暂停后恢复**：暂停期间产生的日志会被缓存，恢复后一次性输出

### 工具栏按钮

| 按钮 | 功能 |
|------|------|
| 清空 | 清空当前日志显示区（同 `clear` 命令） |
| ⏸ 暂停 / ▶ 继续 | 暂停或恢复日志实时刷新 |
| 自动滚动 | 新日志是否自动滚动到底部 |
| 级别: INFO ▼ | 按日志级别过滤显示 |
| 📤 导出日志 | 导出当前会话日志到文件 |
| 🗑 清理旧日志 | 手动清理超过 7 天的旧日志 |
