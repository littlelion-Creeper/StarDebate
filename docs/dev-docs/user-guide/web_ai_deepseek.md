# Web AI：DeepSeek 网页版接入说明

> **非官方 API 方案**，请仔细阅读全文后再使用。

---

## 一、概述

**Web AI** 是 StarDebate 桌面版的一项特殊 AI 功能。它**不直接调用 DeepSeek API**，而是通过内置的 **Playwright** 自动化框架启动 Chromium 浏览器，操控 [chat.deepseek.com](https://chat.deepseek.com) 网页版来完成 AI 对话。

简单来说，这是一种"曲线救国"的方案——让软件代替你打开浏览器、输入问题、等待回复并提取结果。

### 适用场景

- 没有 API Key 时体验 AI 功能的临时替代方案
- 应急使用

---

## 二、免责声明（请务必阅读）

### ⚠ 速度慢且不稳定

- 每次请求需启动 Chromium 浏览器 + 加载网页，速度远慢于 API 模式
- 依赖 DeepSeek 网页结构，可能随时因网页改版而失效
- **建议优先使用 API 模式**

### ⚠ 可能导致软件卡顿或崩溃

- 调用时需运行 Chromium 浏览器，CPU 和内存资源占用较大
- 目前软件仍在对此功能进行优化
- 操作过程有**小概率导致软件卡顿或崩溃**
- 请谨慎使用，重要操作前建议先保存当前工作进度
- 等待时间过长属于正常现象（非卡死），每次请求都需启动浏览器并等待 AI 生成

### ⚠ 数据安全声明

- 登录信息（Cookies / Session）仅保存在本地
- 存储路径：`config/web_ai_sessions/deepseek_state.json`
- **本软件不对任何数据泄露负责**
- 请用户自行保护本地文件安全
- 请妥善保管 DeepSeek 账号密码

## 三、支持的功能范围

### ✅ 支持的功能

以下功能在 AI 来源设为"网页版"后将自动使用 Web AI 调用：

| 功能 | 说明 |
|------|------|
| AI 写稿 | 自动生成一辩稿 |
| AI 扩写 | 基于已有内容扩展论据 |
| AI 分析 | 分析辩稿结构与论点 |
| AI 框架生成 | 生成辩论思维导图 |
| AI 结构分析 | 拆解稿件章节层次 |
| 模拟质询 | AI 扮演对手发起质询 |
| 模拟接质 | AI 模拟对手连续提问 |
| 快速刷题 | 训练辩论技巧 |
| 练习题 | 辩论练习题生成 |
| 资料池 AI 总结导出 | 资料池文件 AI 总结 |

### ❌ 不支持长对话

- 每次 AI 请求都是独立的"一问一答"
- **不支持多轮对话上下文记忆**
- **不支持流式输出（逐字显示）**
- 原因：Web AI 每次调用都会重新启动浏览器并新建对话，无法保持上下文

### ❓ 插件中的 AI 调用

- 第三方插件会通过软件的统一 AI 路由调用 Web AI
- 但由于插件环境差异较大，**是否正常运行需要用户自行判断**
- 如果插件对延迟敏感或需要长对话/流式输出，Web AI 无法满足
- 插件开发者应在各自的文档中注明对 Web AI 的支持情况

## 四、前置依赖

| 依赖 | 状态 | 说明 |
|------|------|------|
| **Playwright** | ✅ 已内置 | 已包含在软件中，无需额外安装 |
| **Chromium 浏览器** | ⚠ 需首次安装 | ~130MB，安装一次后永久可用 |
| **DeepSeek 网页版账号** | 需自行注册 | [chat.deepseek.com](https://chat.deepseek.com)（免费） |

### 安装 Chromium

在设置页 → API 配置 → 查看「Chromium 状态」，如果显示「未安装」，点击「安装 Chromium」按钮一键下载。下载完成后状态会变为「已安装」。

## 五、在软件中的使用方法

### 步骤 1：选择 AI 来源

打开 **设置 → API 配置**，在「AI 来源」中选择一种模式：

| 模式 | 说明 |
|------|------|
| **自动检测**（推荐） | 有 API Key → 自动使用 API 模式（稳定、快速）<br>无 API Key → 自动回退到网页版模式 |
| **API 模式** | 强制使用 API Key 连接 AI 服务 |
| **网页版** | 强制使用 Web AI（无需 API Key） |

### 步骤 2：安装 Chromium（仅首次使用）

1. 设置 → API 配置 → 查看「Chromium 状态」
2. 显示「未安装」→ 点击「安装 Chromium」按钮
3. 等待下载完成（~130MB，视网速 1-5 分钟）
4. 安装完成后状态变为「已安装」

### 步骤 3：登录 DeepSeek 网页版（每次启动软件后）

**方法一：通过设置页登录**

1. 设置 → API 配置 → 找到「DeepSeek 网页版」卡片
2. 点击「登录 DeepSeek 网页版」
3. 弹出浏览器窗口 → 在网页中手动输入账号密码登录
4. 登录成功后浏览器自动关闭，状态变为「已登录」

**方法二：通过标题栏快捷登录**

1. 标题栏右侧找到「登录网页 DeepSeek」按钮
2. 点击后自动执行登录流程
3. 登录后按钮文字变为「已登录（可点击登出）」

### 步骤 4：开始使用 AI 功能

登录成功后，所有支持的 AI 功能将自动使用 Web AI 模式，无需额外操作。

可根据需要前往设置调整：
- **超时时间**（默认 60s）：每次请求的最长等待时间
- **重试次数**（默认 2 次）：超时后自动重试的次数

## 六、为什么每次启动都要重新登录？

当前处于优化阶段，session 持久化逻辑尚不完善。每次软件重启后需要重新登录一次。

整个重新登录过程约 10-30 秒，操作步骤：

1. 打开设置 → API 配置
2. 点击「登录 DeepSeek 网页版」
3. 在弹出的浏览器中完成登录
4. 浏览器自动关闭，状态变为「已登录」

## 七、工作原理（技术细节）

### 整体架构

```
┌──────────┐     ┌──────────┐     ┌──────────────────────┐
│ StarDebate│────▶│ WebAIMgr │────▶│ Playwright (Headless) │
│  AI调用   │     │  单例    │     │    Chromium           │
└──────────┘     └──────────┘     └──────────┬───────────┘
                                              │
                                              ▼
                                   ┌────────────────────┐
                                   │ chat.deepseek.com  │
                                   │     网页版          │
                                   └────────────────────┘
```

### 自动检测逻辑

当 AI 来源设为「自动检测」时，`api_helper.py` 中的路由逻辑如下：

- 配置中有 API Key（非空）→ `provider_type = "api"`，走 API 模式
- 配置中无 API Key → `provider_type = "web"`，回退到 Web AI 模式

### 登录流程

1. 启动**可见**的 Chromium 浏览器窗口 → 打开 chat.deepseek.com
2. 用户在浏览器中手动输入账号密码完成登录
3. 软件检测到聊天输入框出现 → 保存 session（cookies）到本地文件
4. 自动关闭浏览器

### 调用流程（每次 AI 请求）

1. 启动 **headless**（无界面）Chromium
2. 加载之前保存的 session → 访问 chat.deepseek.com
3. 检测 session 是否过期 → 过期则抛出 `SessionExpiredError`
4. 找到输入框 → 填入用户的问题
5. JavaScript 精确定位发送按钮的 SVG 图标 → 模拟点击发送
6. 轮询检测停止生成按钮（SVG 图标消失 = 生成完毕）
7. 内容稳定性检测（连续 3 次检测到相同内容 → 认为回复完成）
8. 提取回复文本 → 关闭浏览器 → 返回结果

### 串行队列

同一时间只能处理 1 个 Web AI 请求。后续请求会排队等待，状态栏会显示当前排队位置。

### 重试机制

- **超时** → 自动重试（默认最多 2 次）
- **Session 过期** → 直接报错，不重试（需重新登录）
- **网页结构变化** → 直接报错，不重试（需等待软件更新）

## 九、常见问题

**Q：提示「找不到输入框」**
A：DeepSeek 网页结构已发生变化，请等待软件更新。

**Q：提示「Session 已过期」**
A：重新登录即可（每次重启后都需要重新登录）。

**Q：提示「请先安装 Chromium」**
A：在设置页中点击「安装 Chromium」，等待下载完成即可。

**Q：AI 等待时间过长，软件是不是卡死了？**
A：等待时间长属于正常现象（每次需启动 Chrome + 加载网页 + 等待 AI 生成）。观察状态栏提示，如果显示「Web AI 调用中…」说明正在正常运行。如果频繁超时可以尝试将超时时间调大（如 120s）。

**Q：使用 Web AI 时软件卡顿或崩溃了**
A：这是已知问题，Chromium 资源占用较大。建议使用前先保存当前工作进度。如果频繁出现，建议切换到 API 模式。

**Q：我安装的插件能用 Web AI 吗？**
A：插件会通过统一的路由调用 Web AI。但由于插件环境差异较大，是否兼容需要用户自行判断。如果插件需要长对话或流式输出，Web AI 无法满足。

**Q：自动检测模式下，我想强制使用 API 模式**
A：在 AI 来源选择中手动切换为「API 模式」即可。
