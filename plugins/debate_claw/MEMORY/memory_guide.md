# DebateClaw AI 记忆系统指南

你拥有一个强大的持久化记忆系统，结合 **Markdown 主题分文件存储** 和 **语义向量检索**。记忆在对话之间保留，支持跨会话的长期记忆、对话历史回溯和资料文档关联。

## 三种记忆方式

### 方式一：快捷标记（推荐日常使用）

在你的回复中使用标记写入/读取：

**写入（推荐）：**
```
[MEM:write user_preferences:名字=张三]
[MEM:write debate_topics:辩题=人工智能伦理]
[MEM:write 新话题:key=value]
```
格式为 `[MEM:write 主题名:key=value]`，主题名和 key 中不能包含 `=` 符号。

- 主题名如果匹配预设映射表（见表），写入对应文件
- 主题名不在表中时，自动创建新文件（降为小写+下划线），并自动添加 `# 标题`
- **向后兼容**：`[MEM:write key=value]` 不带主题名时仍写入 `quick_notes.md`

**读取：**
```
[MEM:read 用户名字]
```

标记会被系统移除，**不会显示给用户**。

### 方式二：语义搜索工具（推荐复杂查询）

使用 `search_memory` 工具在全部记忆中做语义检索：
```
search_memory(query="用户偏好")
```
返回最相关的 Top-5 记忆片段及其来源和匹配度。

### 方式三：文件操作（精确读写）

使用 `file_read` / `file_write` 直接操作 Markdown 记忆文件：
- 记忆文件位于 `MEMORY/topics/` 目录
- 预设主题文件：`user_preferences.md`、`debate_topics.md`、`project_notes.md`、`quick_notes.md`
- 也可创建新文件

## 记忆类型

| 类型 | 存储位置 | 说明 |
|------|----------|------|
| 快捷笔记 | `topics/quick_notes.md` | [MEM:write] 标记自动写入 |
| 用户偏好 | `topics/user_preferences.md` | 偏好、习惯、风格 |
| 辩论话题 | `topics/debate_topics.md` | 辩题、立场、论点 |
| 项目笔记 | `topics/project_notes.md` | 项目进度、待办 |
| 对话历史 | 向量索引库 | 每次对话结束后自动切片索引 |
| 资料文档 | 向量索引库 | 导入的资料文件自动切片索引 |

## 向量检索工作流

1. 你提问或用户发消息时，系统自动从向量库中检索相关记忆片段
2. 这些片段会注入到你的上下文中，帮助你做出更精准的回答
3. 你也可以主动调用 `search_memory` 搜索特定内容

## 项目文件与资料池

每次对话开始时，系统会自动注入以下信息到你的上下文中，**无需任何工具调用**：

1. **当前项目路径** — 你正在工作的辩论赛项目目录
2. **项目可读文件清单** — 目录下所有 `.md` / `.txt` / `.json` 等可读文件（不含权限验证）
3. **资料池文件清单** — 已导入资料池的所有文件

看到文件名后，如需查看文件内容，使用 `file_read(path)` 工具即可（读操作自动通过）。

示例：看到项目中存在 `一辩稿.md` 和 `开篇立论.txt`，你可以直接 `file_read("一辩稿.md")` 读取内容。

## 最佳实践

- 在对话**末尾**写入重要信息，避免中断回复流畅度
- key 使用语义化命名：`用户名字`、`当前辩题`、`已分析论点`
- 不要写入过长内容（建议每条 value ≤ 500 字）
- 优先用 `search_memory` 语义搜索，而非逐个 `[MEM:read]`
- 对话中的重要结论可以总结后写入主题文件，方便后续检索
