辩论框架

双层加密文件格式（AES + 用户密码），安全存储完整辩论数据。密码通过 Windows DPAPI 保护。

系统架构

模块化设计，独立日志进程，安全可靠

📄 启动层

  • StarDebate.py — 启动器，~100 行
  • 启动 LogService 独立进程
  • 创建 QApplication + 主窗口
  • 全生命周期编排 & 清理

🪟 主窗口层

  • StarDebate_app.py — 管理器实例化
  • UIAssemblyMixin — UI 面板组装
  • GlueCodeMixin — 跨模块胶水代码
  • 17 个功能管理器统一调度

📝 日志服务层

  • 独立 multiprocessing.Process
  • SystemHealthMonitor 健康监视
  • 主进程崩溃不影响日志写入
  • 崩溃自动分析（8 维度）
# 三文件架构 StarDebate.py (启动器) ├── LogService 子进程 (star_debate_log.py) │ ├── LogManager → 文件写入 │ └── SystemHealthMonitor → 主进程存活检测 └── StarDebateApp (主窗口) ├── LogClient → 轻量队列投递 ├── UIAssemblyMixin → UI面板组装 ├── GlueCodeMixin → 跨模块桥接 └── 23 个功能模块管理器

技术亮点

PyQt5 桌面技术与现代化工程实践

🔒 API 配置透明加密

api_config.json 通过 Fernet (AES-128-CBC + HMAC-SHA256) 自动加密存储，密钥文件 Windows DPAPI 保护，所有消费者零感知。

📜 起居注系统

运行时 monkey-patch 自动记录功能/插件/API/AI 执行成功/失败，带嵌套上下文栈、错误传播、崩溃自动分析。

🧩 插件系统

完整的插件生命周期管理（加载/启用/禁用/删除），通过 PluginSafeAPI 提供安全沙箱，支持自定义快捷键、SVG 图标、导航栏注册。

⚡ 异步 Worker 模式

所有 AI 功能使用 QThread Worker + pyqtSignal 异步回调，AILoadingBar 引用计数管理多并行任务，界面永不阻塞。

🎨 SVG 渲染器

通用组件 SvgRenderer 支持单色/双色/原生三种渲染模式，颜色嵌入 theme.json 自动适配，LRU 缓存加速，支持插件自定义图标。

⌨️ 快捷键系统

全局快捷键管理器，18 个内置快捷键（5 分类），插件可注册，设置页录制修改，冲突检测，配置持久化。

🛡️ 崩溃容灾

独立日志进程 + 守护日志环 + Qt message handler + Worker 线程保护，崩溃时自动分析生成 8 维度诊断报告。

📦 数据加密存储

.stardebate 双层加密文件格式，辩论数据（辩稿/框架/质询/便签）统一打包，密码通过 Windows DPAPI 绑定当前用户。
